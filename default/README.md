# Login Automation Solution

## Overview
Automated login validation using Python, pytest, and requests. Credentials are securely managed via environment variables for robust CI/CD integration.

## Features
- Secure credential handling
- Pytest-based test execution
- Robust error handling and logging
- Easy CI/CD integration

## Setup Instructions
1. **Clone the repository:**
   ```bash
   git clone https://github.com/lakshmi1212/AgenticAI.git
   cd AgenticAI
   ```
2. **Install Python 3.11** (recommended for compatibility).
3. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```
4. **Configure environment variables:**
   - Copy `.env.template` to `.env` and fill in your credentials.
   - Or set variables in your CI/CD environment:
     - LOGIN_URL
     - LOGIN_EMAIL
     - LOGIN_PASSWORD

## Running the Test
```bash
pytest Tests/login_test.py --junitxml=Tests/login_test_results.xml
```

## Test Results
- Results are output to `Tests/login_test_results.xml` (JUnit XML).
- Check logs for detailed error info.

## Maintenance & Extensibility
- Update credentials in your `.env` or CI/CD secrets.
- Extend `Tests/login_test.py` for additional test cases.
- Review code for PEP8 compliance and security best practices.

## CI/CD Integration
- Recommended for GitHub Actions (see `.github/workflows/ci.yml`).
- Ensure secrets are set in your CI pipeline.

## Files
- `Tests/login_test.py` — Main login test script
- `.env.template` — Environment variable template
- `requirements.txt` — Dependency list
- `README.md` — This documentation
- `login_test.metadata.json` — Test metadata for automation
- `.github/workflows/ci.yml` — CI pipeline (generated by Agent 2)
